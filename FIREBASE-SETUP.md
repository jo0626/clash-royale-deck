# Firebase ログイン セットアップ手順

CR Deck Builder にGoogleログイン＆ユーザーデータ（デッキ保存・CR ID・寄付グレード）を付けるための準備手順です。
**コードは実装済み**なので、あなたがやるのは「Firebaseプロジェクトを作って設定を貼る」だけです。所要10〜15分。

---

## 1. Firebaseプロジェクトを作る
1. https://console.firebase.google.com/ にGoogleアカウントでアクセス
2. 「プロジェクトを追加」→ 名前を入力（例 `cr-deck-builder`）→ 作成
   （Googleアナリティクスは任意。オフでもOK）

## 2. ウェブアプリを登録して設定を取得
1. プロジェクト画面の上部にある **「</>」(ウェブ)** アイコンをクリック
2. アプリのニックネーム（例 `web`）を入力 →「アプリを登録」
3. 表示される `firebaseConfig = { ... }` の中身をコピー
4. このフォルダの **`firebase-config.js`** を開き、`{ ... }` の中身を貼り替える
   （apiKey / authDomain / projectId などを実際の値に）

> apiKey はウェブ用の公開キーなので、サイトに含めて問題ありません（Firebaseの仕様）。
> 実際の保護は後述の「セキュリティルール」で行います。

## 3. ログイン方法を有効化（Google ＋ メール/パスワード）
1. 左メニュー **「構築 > Authentication」** →「始める」
2. 「Sign-in method」タブで以下の2つを有効化：
   - **Google** を選択 →「有効にする」→ サポートメール（あなたのGmail）を選んで保存
   - **メール/パスワード** を選択 →「有効にする」→ 保存
     （「メールリンク（パスワードなし）」はオフのままでOK）

> これでGoogleログインと、メールアドレス＋パスワードでの新規登録・ログイン・
> パスワード再設定（再設定メール送信）が使えるようになります。

## 4. Firestore（データベース）を作る
1. 左メニュー **「構築 > Firestore Database」** →「データベースの作成」
2. リージョンは `asia-northeast1`（東京）推奨 → 次へ
3. まずは **「テストモードで開始」** でOK（後で下のルールに差し替え）
4. 作成

### セキュリティルール（公開前に必ず差し替え）
このフォルダの **`firestore.rules`** の中身を、Firestoreの「ルール」タブに貼り付けて「公開」。

ポイント：このルールは **tier（寄付グレード）をクライアントから変更できない** ようにしてあります。
利用者が自分で `gold` などに書き換えられないので、寄付グレードは運営（コンソール）／将来のサーバーからのみ更新します。
（`crTag` やデッキ保存は本人が自由に編集OK）

## 5. 公開ドメインを許可
本番URL（`crdeckbuilders.com`）でログインさせるには：
1. **Authentication > Settings > 承認済みドメイン** に `crdeckbuilders.com` を追加
   （`localhost` は最初から入っています）

---

## これで完了
`firebase-config.js` に設定を貼れば、ヘッダー右上に「🔑 ログイン」ボタンが出て、
Googleでログイン→デッキ保存・CR ID登録ができるようになります。

## 他ページにもログインを出すには
`strategy.html` / `support.html` / `contact.html` の **`</body>` の直前**に1行追加するだけ：

```html
<script type="module" src="auth.js"></script>
```

（デッキ保存ボタンは `index.html` だけに出ます。他ページはログイン/ログアウトとマイデッキ閲覧用です）

---

## 次にやれること（土台はもうある）
- **寄付グレードで着せ替え**：各ユーザーのプロフィールに `tier`（free/bronze/silver/gold）があります。`auth.js` の `TIERS` がグレード定義。寄付確認後にこの値を上げると、グレードに応じてテーマ/見た目を出し分けられます（テーマ切替の実装は次フェーズ）。
- **CRユーザーID連携**：`crTag` に保存済み。Supercell公式API（要プロキシ）と繋げば戦績表示なども可能。
- **デッキ保存**：`users/{uid}/decks` に保存され、マイデッキからワンタップ読込できます。
